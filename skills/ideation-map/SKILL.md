# Ideation Map: 可能性地图

用户有一个模糊的方向，但还不确定该研究什么。你的工作是**铺开知识地图**，让用户看到可能性的全貌，然后用她的跨域直觉做连接。

**核心公式**：你的知识广度 × 用户的泛化能力 = 单独都产生不了的方向。

## 什么时候用

- 用户说"我想做个 xxx"但还没有具体 spec
- 用户说"有个想法"/"帮我想想"
- 在 brainstorming 之前，需要先看清全局
- 编排层判断用户意图是探索阶段时

## 与 brainstorming 的区别

| | ideation-map | brainstorming |
|--|-------------|---------------|
| **目的** | 铺开可能性，发现方向 | 细化方向，产出 spec |
| **输出** | 可能性地图文件 | 设计文档 |
| **用户角色** | 做跨域连接、选方向 | 回答细化问题、确认设计 |
| **深度** | 广而浅 | 窄而深 |
| **后续** | → brainstorming 或 → research | → writing-plans |

## 执行流程

```
用户给出模糊方向
    ↓
Step 1: 理解原始意图（1-2 个澄清问题，不超过）
    ↓
Step 2: 知识扫描（相关领域、已有方案、类似项目）
    ↓
Step 3: 模式提取（跨领域可借鉴的模式）
    ↓
Step 4: 产出可能性地图文件
    ↓
Step 5: 等用户做结构映射
    ↓
用户选择方向 → 进入 brainstorming 或 research
```

## Step 1: 理解原始意图

**最多问 2 个问题**，不要变成 brainstorming 的 20 问：

1. "你想解决什么问题？"（如果从用户的话里看不出来）
2. "有没有看到过类似的东西让你有这个想法？"（追溯灵感来源）

如果用户的方向已经清晰，直接进 Step 2，不问。

## Step 2: 知识扫描

用你的知识广度，从**至少 3 个维度**扫描：

### 扫描维度（按相关性选 3-5 个）

- **同类方案**：市面上谁在做类似的事？怎么做的？
- **上游领域**：这个问题在更抽象的层面已经被解决过了吗？
- **邻域借鉴**：其他行业/领域有没有结构相似的解法？
- **技术栈**：实现这个有哪些技术路线？各自适合什么场景？
- **历史演化**：这个问题的解法经历过哪些代际变化？
- **反面案例**：谁做过类似的但失败了？为什么？

### 扫描深度控制

每个维度：**2-3 句话概述 + 1 个关键 insight**。不展开，只勾勒轮廓。
用户看完觉得某个方向有意思，再在 brainstorming/research 里深挖。

## Step 3: 模式提取

从扫描结果中，提取**跨领域可复用的模式**：

```markdown
## 可借鉴模式

### 模式 1: [模式名]
- 来源: [哪个领域/项目]
- 核心逻辑: [一句话]
- 可能适用于: [跟用户方向的连接点]

### 模式 2: ...
```

这些模式是给用户做"结构映射"用的。用户看到这些，可能会说"模式 2 和那个邻域的做法能连起来"——这种跨域连接是她最擅长的。

## Step 4: 产出可能性地图文件

写入文件 `{project-dir}/ideation-map.md`，**不在主对话展开**：

```markdown
# Ideation Map: [项目/方向名]

## 原始方向
[用户的一句话描述]

## 领域扫描

### 1. [维度名]
[2-3 句概述]
**关键 insight**: [一句话]

### 2. [维度名]
[2-3 句概述]
**关键 insight**: [一句话]

### 3. [维度名]
...

## 可借鉴模式

### 模式 1: [模式名]
- **来源**: [领域/项目]
- **核心逻辑**: [一句话]
- **连接点**: [跟用户方向怎么接]

### 模式 2: ...

## 等你的判断

以下连接点需要你的直觉：
1. [维度A] 和 [维度C] 的做法有结构相似性，值得探索吗？
2. [模式2] 能不能用在 [用户场景] 里？
3. 你看到了什么我没看到的跨域关联？

## 可能的方向（供参考，不是结论）
- **方向 A**: [一句话] — 如果你在意 [X]
- **方向 B**: [一句话] — 如果你在意 [Y]
- **方向 C**: [一句话] — 如果你在意 [Z]
```

## Step 5: 呈现给用户

主对话只说：

```
铺了一张可能性地图，扫了 [N] 个维度，提取了 [M] 个可借鉴模式。

几个有意思的点：
- [最有趣的 insight 1]
- [最有趣的 insight 2]

详细在 [文件路径]。你看看哪些方向值得深挖，或者你看到了什么跨域连接我没想到的？
```

然后**等用户回应**。不催，不替用户做判断。

## 用户回应后的分流

- 用户选了方向 → 进入 `brainstorming`（细化为 spec）
- 用户发现了跨域连接 → 更新 ideation-map，可能再扫一轮
- 用户说"需要更多信息" → 进入 `research`（深度研究某个维度）
- 用户说"我再想想" → 存到 RecallNest，下次继续

## 与流水线的集成

```
用户说"我想做个 xxx"
    ↓ 编排层判断：探索阶段
ideation-map（铺可能性）
    ↓ 用户选方向
brainstorming（细化 spec）或 research（深挖某维度）
    ↓
writing-plans → multi-role-review → execution ...
```

## 关键原则

- **广而浅，不深挖** — 这不是 research，是扫描
- **给材料不给结论** — 结论是用户做的
- **为跨域连接留空间** — 模式提取就是给用户的"连接素材"
- **不催不抢** — 等用户的直觉，不替她做判断
- **写文件不占上下文** — 地图写文件，主对话只给 highlights
