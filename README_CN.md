# Workflow Orchestrator

**Claude Code 的自然语言流水线编排层。**

说人话，AI 自动判断你在哪个阶段，调用对应的能力。从构思到上线，全程不需要记任何斜杠命令。

## 工作原理

```
你说："我想做一个缓存层"

                    +-------------------+
                    |    你说人话        |
                    +---------+---------+
                              |
                              v
                    +---------+---------+
                    |   意图识别         |
                    |  (编排层)          |
                    +---------+---------+
                              |
          +-------------------+-------------------+
          |           |           |           |
          v           v           v           v
     +---------+ +---------+ +---------+ +---------+
     | 构思    | | 规划    | | 审查    | | 执行    |
     | (地图)  | | (计划)  | | (4角色) | | (代码)  |
     +---------+ +---------+ +---------+ +---------+
          |           |           |           |
          +-------------------+-------------------+
                              |
                              v
                    +---------+---------+
                    |   发布 + 知识沉淀  |
                    +-------------------+
```

### 流水线阶段

| 阶段 | 做什么 | 触发方式 |
|------|--------|---------|
| **构思** | 扫描多个领域的可能性，输出方向地图 | "我想做..." / "有个想法" |
| **研究** | 深入用户选定的方向 | 用户从地图中选方向 |
| **规划** | 把设计方案拆成实现步骤 | "帮我规划一下" |
| **审查** | 4 个子 agent 从不同角度并行审查计划 | 规划完成后自动触发 |
| **执行** | 子 agent 并行实现代码 | 用户确认审查结果后 |
| **发布** | 合并、测试、上线 | "发布吧" |
| **沉淀** | 把经验记录下来，下次可复用 | 发布完成后自动触发 |

**核心规则：** 产出型步骤自动流转，决策型步骤等你拍板。

## 包含哪些能力

### 1. 流水线编排层 (`/workflow-orchestrator`)

路由层。解析你的自然语言，判断当前阶段，调度对应能力。支持阶段跳转（"不用构思了，直接规划"）和回溯（"方向不对，重来"）。

### 2. 多角色审查 (`/multi-role-review`)

4 个独立子 agent 并行审查你的计划，各自关注不同维度：

| 角色 | 关注点 |
|------|--------|
| **用户代言人** | 这真的解决了用户的问题吗？有没有过度工程？ |
| **架构师** | 边界清晰吗？抽象合理吗？可扩展吗？ |
| **风险猎人** | 什么会出错？安全隐患？依赖风险？ |
| **务实主义者** | 有更简单的方案吗？是不是 YAGNI 了？ |

审查完成后，系统交叉比对盲区，提取核心张力——需要你做判断的权衡点。这个模式借鉴了 [content-alchemy](https://github.com/AliceLJY/content-alchemy) 的三方质询设计。

### 3. 可能性地图 (`/ideation-map`)

正式研究之前的"元研究"。当你有模糊方向但不确定该深挖什么时，这个能力会：

1. 扫描 3-5 个相关领域、已有方案、邻域做法
2. 提取跨领域可复用的模式
3. 输出结构化的"可能性地图"，标注连接点
4. 等你做跨域关联

逻辑：AI 有知识广度，你有跨域直觉。合在一起能找到各自单独找不到的方向。

## 安装

```bash
git clone https://github.com/AliceLJY/workflow-orchestrator.git
cd workflow-orchestrator
bash install.sh
```

三个 skill 会被复制到 `~/.claude/skills/`。重启 Claude Code 后生效。

## 使用示例

不需要直接调用 skill，直接说话：

```
你：  "我想给 app 做一个通知系统"
  --> 编排层路由到可能性地图
  --> 你拿到 4 个领域的扫描结果
  --> 你选一个方向

你：  "方案 B 不错，细化一下"
  --> 路由到头脑风暴，然后规划
  --> 计划写好

      （自动触发多角色审查）
  --> 4 个审查员并行工作
  --> 你看到："计划没大问题，2 个点需要你拍板"

你：  "开干"
  --> 路由到执行阶段
  --> 子 agent 并行实现

你：  "发布吧"
  --> 合并、测试、上线
  --> 经验自动沉淀
```

## 使用前 vs 使用后

**使用前（手动管理 skill）：**
```
> /brainstorm notification system       # 需要知道命令
> /write-plan                           # 需要记住顺序
> /multi-role-review                    # 需要知道这个存在
> /execute-plan                         # 需要手动触发
> /code-review                          # 容易忘
> /finish                               # 希望你还记得
```

**使用后（自然语言编排）：**
```
> 我想做一个通知系统
  ...（一切从对话中自然流转）...
> 发布吧
```

## 设计原则

- **零命令记忆** -- 用户永远不需要学斜杠命令
- **上下文隔离** -- 子 agent 拿到精准构造的上下文，不继承完整对话历史
- **人在回路** -- 产出型步骤自动推进，决策点必须等用户确认
- **优雅降级** -- 缺少某个 skill 时回退到手动模式，不卡住流程
- **可恢复** -- 中途切换窗口，回来可以接着做

## 项目结构

```

install.sh                       # 一键安装脚本
docs/
  research.md                    # 设计研究笔记
  plan.md                        # 实现计划
```

## AliceLJY Claude Code 生态

本项目与以下项目协同工作：

- [RecallNest](https://github.com/AliceLJY/recallnest) -- 基于 LanceDB 的 Claude Code 长期记忆
- [content-alchemy](https://github.com/AliceLJY/content-alchemy) -- 多视角审查的内容创作流水线

本项目的多角色审查模式直接借鉴了 content-alchemy 的三方质询设计。RecallNest 提供检查点/恢复能力，让流水线可以跨会话延续。
